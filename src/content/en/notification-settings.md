# Notification settings

Use this page to manage how the system contacts you about important events.

## Page purpose

- Configure notification email behavior.
- Set event subscriptions and thresholds.
- Manage webhook delivery targets.
- Review delivery history.

## What you will see

### Email settings

- Notification email preferences.
- Test-email action.
- Cooldown or sending-frequency controls.

### Event subscriptions

- Event list with enable or disable controls.
- Threshold fields for alerts that depend on counts or limits.

### Webhooks and logs

- Webhook list.
- Add, edit, and test webhook actions.
- Delivery log table with pagination.

## Common actions

1. Confirm the email address that should receive alerts.
2. Enable only the events you actually need.
3. Add webhooks for automation or external monitoring tools.
4. Use test actions before relying on a new delivery target.
5. Review delivery logs when alerts do not arrive as expected.

## Notes

- Webhooks should point to endpoints you control.
- Notification fatigue is reduced by choosing realistic thresholds.
- Delivery logs help separate content issues from destination issues.
