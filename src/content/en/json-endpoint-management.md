# JSON endpoint management

Use this page to publish accessible JSON content endpoints.

## Page purpose

- Create JSON endpoints with a name and slug.
- Control whether access is public or password-protected.
- Copy endpoint URLs.
- Preview or inspect JSON payloads.

## What you will see

### Endpoint list

- Name, slug, and description.
- Public or password-protected state.
- Access count.

### Management actions

- Create endpoint.
- Edit endpoint.
- Delete endpoint.
- Copy URL.
- View JSON content.

## Common actions

1. Create the endpoint and fill in a short description.
2. Decide whether the endpoint should be public or protected.
3. Enter or paste the JSON content.
4. Save and copy the URL for the intended consumers.
5. Re-open the viewer when you need to confirm the published output.

## Notes

- Password protection is useful when content is shareable but not fully public.
- Existing links usually remain valid unless the slug changes.
- Pair this page with article or API documentation when users need usage guidance.
