# IP 监控面板

此页面用于查看 IP 风险数据、黑名单状态与白名单控制。

## 页面用途

- 查询指定 IP 的当前状态和风险等级。
- 查看错误权重、衰减表现和封禁等级。
- 维护黑名单与白名单记录。
- 重置或手动调整 IP 错误权重。

## 你会看到什么

### 总览与查询工具

- 按 IP 地址查询当前状态。
- **封禁状态与等级**：系统根据累积错误权重自动将 IP 分入三个等级：
  - **一级**——短时软封禁。
  - **二级**——中等时长封禁。
  - **三级**——长时间硬封禁。
- 当前错误权重、各等级的触发阈值、当前封禁的剩余时间。
- 错误衰减估计：显示权重在无新错误的情况下预计何时跌破各阈值。
- 错误类型明细（每类错误的权重可在服务端配置中调整）。

### 黑名单管理

- 已封禁 IP 列表，含封禁原因、添加人和到期时间。
- 搜索、新增、编辑和删除（解封）操作。
- 所有写入操作需通过 **二次验证步骤（step-up）**。

### 白名单管理

- 已豁免 IP 列表（白名单 IP 不会被自动封禁）。
- 新增和移除操作。
- 所有写入操作需通过 **二次验证步骤（step-up）**。

## 错误权重与衰减机制

错误权重是封禁系统的核心：

- 每类错误（如登录失败、无效令牌、频繁请求等）携带可配置的权重值。
- IP 的累积权重超过对应等级阈值后，进入相应封禁级别。
- **错误衰减**会随时间自动降低权重。衰减速率、最小阈值和衰减间隔均可在**服务端配置 → IP 封禁**区块中调整。
- 详情页的衰减估计显示在无新错误的情况下，该 IP 何时可以自然解封。

如需查看或调整单个 IP 的错误权重明细，可使用当前页面的权重编辑器，或调用 `GET /ip-blacklist/error-status/{ip}`；重置权重用 `DELETE /ip-blacklist/error-status/{ip}`。

## 常见操作

1. 先查询可疑 IP，查看其当前等级、权重和封禁剩余时间。
2. 查看错误类型明细，了解该 IP 被标记的原因。
3. 如果封禁是误判，重置错误权重并删除黑名单记录。
4. 如果 IP 确实存在恶意行为，手动创建或编辑黑名单记录并填写原因。
5. 将可信基础设施 IP 添加到白名单，防止其触发自动封禁。
6. 调整服务端配置中的阈值后，重新查询受影响的 IP 确认新等级生效。

## 说明

- 白名单会完全绕过自动封禁逻辑，仅用于你能控制的可信 IP。
- 错误衰减只能被动降低权重；在封禁到期前不会提前解封。
- 三级封禁机制提供了渐进式响应：一级应对短暂异常，三级应对持续滥用。
- 建议与服务端配置（调整阈值和权重）和系统日志（查看该 IP 的请求记录）配合使用。
