# 用户在线监控

此页面用于查看用户在线状态、会话明细和历史时间线，并在必要时执行强制下线。它更偏向运维、客服、安全排障和账号风控场景。

## 页面用途

- 统计当前在线与离线用户概况。
- 按用户名、状态、IP、地域等条件筛选会话。
- 查看单个用户的在线详情和会话时间线。
- 强制某个会话或某个用户下线。
- 导出会话历史为 CSV 或 JSON。

## 你会看到什么

### 顶部统计卡片

总览区通常包含以下指标：

- 在线用户数
- 离线用户数
- 活跃会话数
- 平均会话时长

这些数字适合先帮助你判断问题是局部异常，还是系统范围的会话波动。

### 筛选区

当前页支持的筛选维度包括：

- 关键字
- 在线状态
- IP 地址
- 地域

这意味着你可以按人查，也可以按来源查。例如：

- 想找某个用户最近是否在线
- 想看某个 IP 是否有异常活跃
- 想判断某个地区是否出现批量会话

### 总览列表

列表通常展示：

- 用户名
- 姓名
- 当前状态
- IP 地址
- 地域
- 开始时间
- 最后心跳时间
- 结束时间
- 会话时长

桌面端会以表格展示；移动端则会用卡片方式压缩同样的信息。

### 详情抽屉

点击“查看详情”后，会打开某个用户的详情抽屉。这里通常能看到：

- 当前会话数
- 总会话数
- 总在线时长
- 最新 IP
- 最新地域
- 最新 User-Agent
- 最近开始时间 / 心跳时间 / 结束时间

这一层更适合做单用户排查，而不是大盘筛查。

### 时间线区域

详情抽屉里还会分成两块：

- 当前在线会话卡片
- 历史离线时间线

在线会话卡片更适合处理“现在还在线的连接”；离线时间线则适合追溯历史行为。

## 详情页可执行的动作

### 时间范围过滤

详情区支持时间范围筛选，用于缩小会话历史：

- 查看某个时间窗口内的登录活动
- 排查某次异常发生前后的会话变化
- 只导出一段时间内的记录

### 强制某个 session 下线

在线会话卡片上可以对单个 `sessionId` 执行强制下线。

适用场景：

- 某个终端存在异常登录
- 同一用户的某台设备需要被单独断开
- 想保留其他会话，只终止其中一个连接

### 强制某个用户全部下线

详情工具栏还可以直接让某个用户整体下线。

适用场景：

- 账号被盗用或疑似泄露
- 用户需要立即全部重新登录
- 需要快速终止该用户当前所有活动会话

### 导出会话记录

支持导出：

- CSV
- JSON

导出内容通常包含：

- `sessionId`
- `authSessionId`
- 开始时间、最后心跳、结束时间
- 时长
- IP、地域、User-Agent
- 是否在线

## 强制下线的风险说明

强制下线属于高风险动作，执行前应确认影响范围。

尤其要注意：

- 强制某个 session 下线，只影响对应会话。
- 强制某个用户下线，会影响该用户当前所有活跃会话。
- 如果操作命中了当前操作者自己的登录会话，系统会把当前操作者也登出。

这意味着你在排障时要先看清目标用户是不是自己，或当前浏览器是不是目标会话之一。

## 常见操作

1. 先看顶部统计卡片，确认异常是个体还是批量。
2. 通过关键字、状态、IP 或地域筛选目标记录。
3. 打开目标用户的详情抽屉。
4. 根据在线会话卡片或离线时间线判断问题范围。
5. 需要保留其他连接时，只强制某个 session 下线。
6. 需要完全清场时，再对整个用户执行强制下线。
7. 如需留档或交叉分析，导出 CSV 或 JSON。

## 权限要求

本页通常涉及以下权限：

- `user_online_monitor:read`
- `user_online_monitor:force_offline`

其中：

- `user_online_monitor:read` 用于查看总览、详情和时间线。
- `user_online_monitor:force_offline` 用于执行强制下线动作。

如果只有读取权限而没有强制下线权限，通常只能查看，不能执行高风险操作。

## 说明

- 在线状态依赖会话心跳，不应把它理解成“用户此刻一定正在操作界面”。
- 详情抽屉中的时间线更适合单用户分析，大盘趋势仍建议结合系统统计或日志页。
- 导出的 JSON 更适合程序化处理，CSV 更适合人工查看和表格分析。
