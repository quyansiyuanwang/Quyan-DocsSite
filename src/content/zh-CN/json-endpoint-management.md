# JSON 端点管理

此页面用于发布可访问的 JSON 内容端点。

## 页面用途

- 创建带名称和 slug 的 JSON 端点。
- 管理公开或受密码保护的访问方式。
- 复制访问链接。
- 预览或查看 JSON 内容。

## 你会看到什么

### 端点列表

- 名称、slug、说明。
- 公开或密码保护状态。
- 访问次数。

### 管理动作

- 新建端点。
- 编辑端点。
- 删除端点。
- 复制 URL。
- 查看 JSON 内容。

## 常见操作

1. 先创建端点并填写基础说明。
2. 决定该端点是公开访问还是需要密码保护。
3. 填写或粘贴 JSON 内容。
4. 保存后复制链接给需要的调用方。
5. 如需排错，可再次打开查看器确认输出内容。

## 说明

- 密码保护适合共享但不希望完全公开的场景。
- 修改内容后，旧链接通常仍然可用，关键在于 slug 是否改变。
- 若需要面向用户编写配套说明，可结合文章页或 API 文档页。
