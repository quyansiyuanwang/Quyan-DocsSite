# 消耗统计

此页面用于分析用户、模型、渠道和转发 Token 维度的使用量。

## 页面用途

- 按时间和资源维度筛选使用数据。
- 比较不同渠道、模型和用户的趋势。
- 查看汇总总量和明细表。

## 你会看到什么

### 筛选区域

- 时间范围筛选。
- 用户、模型、渠道、转发 Token 的多选筛选。
- 当前生效筛选标签。

### 汇总与图表

- 汇总卡片。
- 每日趋势图。
- 渠道分布图。
- 模型分布图。
- 用户分布图。

### 明细表

- 渠道表。
- 模型表。
- 用户表。

## 常见操作

1. 先选择与事件或账期对应的时间范围。
2. 再用用户、模型、渠道或 Token 条件缩小范围。
3. 先看汇总卡片，再进入明细表。
4. 对比分布图找出集中点或异常点。

## 说明

- 过多筛选条件可能掩盖整体趋势，当总量异常时应适当放宽条件。
- 此页既适合性能分析，也适合成本排查。
- 如果要处理单个用户的财务修正，应进一步查看余额管理或业务日志。
