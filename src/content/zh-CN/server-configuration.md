# 服务端配置

此页面用于维护全局服务端设置，修改前需通过二次验证步骤确认。

## 页面用途

- 配置注册行为（是否开放注册、每邮箱账号上限、默认群组、验证码有效期）。
- 维护计费设置（充值比率）。
- 管理 SMTP 与邮件发送配置。
- 调整 IP 封禁阈值与封禁时长。
- 配置验证码（Captcha）提供商与信任窗口。
- 调整心跳检测间隔与超时时间。
- 配置中继上游地址与密钥。
- 设置通知系统默认订阅事件与余额阈值。

## 你会看到什么

### 配置区块

| 区块        | 主要参数                                                                       |
| ----------- | ------------------------------------------------------------------------------ |
| **注册**    | 是否开放注册、每邮箱最大账号数、默认群组、验证码有效期（秒）                   |
| **计费**    | 充值比率（`rechargeRatio`）                                                    |
| **SMTP**    | 主机、端口、TLS、发件人账号与名称                                              |
| **验证码**  | 提供商（`recaptcha` / `turnstile` 等）、备用提供商、最低分值、信任窗口（分钟） |
| **心跳**    | 检测间隔秒数、超时秒数                                                         |
| **IP 封禁** | 三级阈值与封禁时长、错误衰减开关与速率                                         |
| **中继**    | 上游 URL 与 API Key                                                            |
| **通知**    | 默认订阅事件列表、余额告警阈值                                                 |

### 公开只读端点

- `GET /config/public/registration` — 前端用于判断是否显示注册入口。
- `GET /config/public/captcha` — 前端用于获取当前 Captcha 提供商及是否启用。

### 编辑工具

- 按区块折叠展示，每个区块独立保存。
- 所有写入操作需通过 **二次验证步骤确认（step-up）**。
- 需要 `SYSTEM_CONFIG` 权限。

## 常见操作

1. 展开需要调整的区块。
2. 只修改当前任务相关的参数。
3. 点击保存后完成二次验证。
4. 重大规则变更后再检查相关监控表现。

## 说明

- 此处的小改动可能影响注册、计费、邮件发送、Captcha 验证和防滥用策略。
- SMTP 配置错误往往在通知或找回密码流程中暴露；修改后建议发送测试邮件确认。
- IP 封禁和衰减调整应结合 IP 监控数据进行。
- 验证码信任窗口（`trustWindowMinutes`）表示通过 Captcha 后多长时间内无需再次验证。
