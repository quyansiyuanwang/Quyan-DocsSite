# 群组管理

此页面用于维护用户群组以及群组级权限集合。所有写入操作需通过二次验证步骤确认。

## 页面用途

- 创建和编辑群组。
- 调整群组名称、等级和说明。
- 配置群组拥有的权限。
- 删除不再使用的群组。

## 你会看到什么

### 群组列表

- 群组名称。
- 等级、说明。
- 权限数量。

### 管理动作

- 新建群组。
- 编辑群组。
- 删除群组。
- 打开权限编辑器。

## 常见操作

1. 创建群组前先明确该群组的职责范围。
2. 使用编辑功能维护群组名称和说明，便于后续识别。
3. 打开权限编辑器批量勾选该群组应拥有的权限。
4. 删除群组前先确认没有用户仍依赖该群组。

## 权限要求

| 操作 | 所需权限 |
|------|----------|
| 查看群组列表 / 详情 | `GROUP_READ` |
| 创建群组 | `GROUP_CREATE` |
| 编辑群组 | `GROUP_UPDATE` |
| 删除群组 | `GROUP_DELETE` |
| 修改群组权限 | `GROUP_PERMISSION_ADD` |

## 层级限制

群组拥有数字等级字段（level），操作受以下规则约束：

- 非超级管理员只能管理**等级高于自身**的群组（即数值更大）。
- 不能创建与自身同级或更高级别（数值更小或相等）的群组。
- 不能修改或删除与自身同级或更高级别的群组。
- 不能将群组等级调整为与自身同级或更高。

## 说明

- 群组权限通常会影响同组所有用户，最终有效权限 = 群组权限 + 用户附加权限 - 用户移除权限。
- 所有写入操作（创建/编辑/删除/权限修改）均需通过**步骤验证（step-up 2FA）**。
- 若用户存在额外权限或移除权限，最终效果应再到权限管理页确认。
