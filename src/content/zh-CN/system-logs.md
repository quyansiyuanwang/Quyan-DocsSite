# 系统日志

此页面用于检查 API 日志和服务端日志文件。

## 页面用途

- 筛选并查看 API 请求日志。
- 检查请求与响应详情（含请求头和响应头）。
- 浏览服务端日志文件并进行内容搜索。
- 查看日志统计图表。
- 支持后端与基础设施问题排查。

## 你会看到什么

### 日志统计面板

- 各状态码（2xx / 4xx / 5xx）的请求量分布。
- 按日期统计的请求趋势图。
- 错误率走势，用于快速发现异常时段。
- 权限要求：`SYSTEM_LOGS_READ` 或 `SYSTEM_LOGS_STATS_READ`。

### API 日志标签页

- 按方法、路径、状态码、用户、请求 ID、IP、日期范围筛选。
- 可展开的日志行，包含：
  - 请求与响应载荷。
  - **请求头与响应头**（自 logging 模块增强后记录）。
  - 请求 ID、IP、User-Agent 等元数据。
- 分页控件。

> **注意**：高频路径（如 `/relay/proxy`）的请求体默认不记录，以节省存储。

### 服务端日志标签页

- 日志文件列表（按类型分类）。
- 文件内容查看器。
- 文件内搜索。
- 行数限制控制。

## 常见操作

1. 先查看统计面板，找出错误率异常的时段。
2. 通过筛选锁定目标请求或时间窗口。
3. 展开日志行查看载荷、请求头和响应头细节。
4. 当问题超出单次请求范围时，转到服务端文件搜索。
5. 跨工具排查时使用请求 ID（`x-request-id`）进行关联。

## 说明

- 请求和响应载荷可能包含敏感运维数据，请谨慎共享。
- 响应头记录便于调试 CORS、缓存、限速等头部设置问题。
- 展开单行适合链路分析，文件搜索更适合查找反复出现的问题模式。
- 这是诊断转发、认证、计费或监控问题的核心页面之一。
