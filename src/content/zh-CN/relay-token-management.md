# Relay Token 管理

此页面用于管理转发 Token、渠道路由、额度窗口和故障转移策略。

## 页面用途

- 创建和维护转发 Token。
- 查看 Token 用量、到期时间与额度窗口。
- 控制渠道顺序与故障转移逻辑。
- 在不立即删除的情况下先停用 Token。
- 批量导入/导出 Token 和渠道配置。

## 你会看到什么

### Token 列表

- Token 名称与可搜索的 Token 信息。
- 请求次数与累计 Token 用量。
- 到期时间、状态与额度使用情况。
- 分页模式或全部展示模式。

### 额度窗口（Quota Windows）

每个 Token 可配置一到多个滚动时间窗口限制，例如每分钟、每小时或每天的最大用量：

- 启用 / 禁用额度窗口总开关。
- 每个窗口独立显示已用量 / 上限与重置时间。
- 达到上限时该 Token 请求会被拒绝，直至窗口重置。

### IP 白名单

- 为 Token 绑定允许访问的 IP 或 CIDR 范围。
- 留空表示不限制来源 IP；填写后仅白名单内的 IP 可使用该 Token。

### 渠道与故障转移

- 有序渠道列表。
- 故障转移摘要（主用渠道不可用时自动切换）。
- 额度窗口详情面板。

### 管理动作

- 新建 Token。
- 编辑 Token（含额度窗口和 IP 白名单）。
- 克隆 Token（复制全部配置生成新 Token）。
- 刷新 Token 值（生成新密钥，旧密钥立即失效）。
- 启用或停用 Token。
- 删除 Token。
- **批量操作**：导入 / 导出 Token 列表（JSON 格式）。
- **渠道批量操作**：导入 / 导出渠道配置。

## 常见操作

1. 创建具有可识别名称的 Token。
2. 配置渠道顺序和故障转移策略。
3. 如需限速，开启额度窗口并设置各时间维度的上限。
4. 如需来源限制，填写 IP 白名单。
5. 定期查看额度使用情况，避免意外限流。
6. 想先停止流量时，可优先停用而不是直接删除。
7. 批量迁移或备份配置时使用导入/导出功能。

## 说明

- 刷新 Token 值会立即使旧 Token 字符串失效，调用方需同步更新。
- 额度窗口与 Token 总配额（`quotaLimit`）是两套独立的限制，可同时启用。
- 批量导入时现有 Token 不会被覆盖，导入内容作为新条目追加。
- "全部展示"模式适合审计，但阅读负担更高。
