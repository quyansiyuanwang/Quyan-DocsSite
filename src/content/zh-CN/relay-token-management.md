# Relay Token 管理

此页面用于管理转发 Token、渠道路由、额度窗口和故障转移策略。

## 页面用途

- 创建和维护转发 Token。
- 查看 Token 用量、到期时间与额度窗口。
- 控制渠道顺序与故障转移逻辑。
- 在不立即删除的情况下先停用 Token。

## 你会看到什么

### Token 列表

- Token 名称与可搜索的 Token 信息。
- 请求次数与累计 Token 用量。
- 到期时间、状态与额度使用情况。
- 分页模式或全部展示模式。

### 渠道与故障转移

- 有序渠道列表。
- 故障转移摘要。
- 额度窗口详情。

### 管理动作

- 新建 Token。
- 编辑 Token。
- 复制 Token。
- 启用或停用 Token。
- 删除 Token。
- 导出或更多操作菜单。

## 常见操作

1. 创建具有可识别名称的 Token。
2. 配置渠道顺序和故障转移策略。
3. 定期查看额度使用情况，避免意外限流。
4. 想先停止流量时，可优先停用而不是直接删除。
5. 仅在确认不再被依赖后再删除 Token。

## 说明

- “全部展示”模式适合审计，但阅读负担更高。
- 额度窗口详情常用于解释短时间内的请求失败。
- 该页面通常需要和余额、上游状态一起联动查看。
