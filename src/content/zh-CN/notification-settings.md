# 通知设置

此页面用于管理系统如何向你发送重要通知，以及管理员配置全局通知默认值。

## 页面用途

- 配置通知邮箱与冷却时间。
- 设置事件订阅与告警阈值。
- 管理 Webhook 投递目标。
- 查看投递历史。
- （管理员）修改全局默认订阅事件和默认余额阈值。

## 你会看到什么

### 邮件设置

- 通知邮箱偏好（可与账号邮箱不同）。
- 发送测试邮件按钮。
- 冷却时间设置（防止短时间内重复通知）。

### 事件订阅

- 事件列表及启用开关。
- 依赖数量或余额的告警阈值设置。
- 新用户注册时默认订阅全部事件（由管理员在服务端配置中设定）。

### Webhook 与投递日志

- Webhook 列表（URL、状态、最后投递时间）。
- 新增、编辑、测试 Webhook 操作。
- 带分页的投递日志，可排查投递失败原因（含状态码与响应体）。

### 管理员：全局通知配置

管理员可通过**服务端配置**页修改以下全局默认值（存储于 `CONFIG_KEYS.NOTIFICATION`）：

| 配置项                    | 含义                     |
| ------------------------- | ------------------------ |
| `defaultSubscribedEvents` | 新用户的默认订阅事件列表 |
| `defaultThresholds`       | 新用户的默认余额告警阈值 |

修改全局默认值不会影响已有用户，仅对此后新创建的账号生效。

## 常见操作

1. 确认接收通知的邮箱地址。
2. 仅开启自己真正需要的事件，避免通知疲劳。
3. 为自动化系统或外部监控工具添加 Webhook。
4. 在正式依赖前先使用测试动作验证目标可用。
5. 通知未送达时先查看投递日志，区分内容问题还是目标端问题。

## 说明

- Webhook 应指向你能控制的目标地址。
- 投递日志记录每次推送的状态码和响应体，便于调试。
- 事件订阅变更立即生效，无需重启服务。
