# 反馈管理

此页面供管理员查看、处理和回复用户提交的反馈（建议、Bug 报告等）。

## 页面用途

- 浏览所有用户反馈。
- 按类型、状态、优先级、负责人筛选。
- 更新反馈状态和优先级。
- 添加内部处理评论。
- 删除不合适的反馈条目。

## 你会看到什么

### 反馈列表

- 反馈标题、类型（建议 / Bug / 其他）、提交用户。
- 工作流状态：`pending`（待处理）、`processing`（处理中）、`accepted`（已采纳）、`rejected`（已拒绝）、`completed`（已完成）。
- 优先级：`low` / `medium` / `high` / `urgent`。
- 负责人（assignee）和提交时间。
- 时间范围筛选（`startTime` / `endTime`）。

### 反馈详情

- 用户提交的原始内容。
- 用户跟进评论。
- **管理员评论区**（内部处理记录，仅管理员可见）。

## 常见操作

1. 按状态 `pending` 筛选，找出尚未处理的新反馈。
2. 打开详情，阅读反馈内容后更新优先级和状态。
3. 在管理员评论区记录处理进展或决策。
4. 将反馈分配给负责人跟进。
5. 处理完毕后将状态更新为 `completed`。

## 权限要求

| 操作                         | 所需权限                 |
| ---------------------------- | ------------------------ |
| 查看反馈列表 / 详情          | `FEEDBACK_REVIEW_READ`   |
| 更新状态 / 优先级 / 添加评论 | `FEEDBACK_REVIEW_UPDATE` |
| 删除反馈                     | `FEEDBACK_REVIEW_UPDATE` |

## 说明

- 用户只能看到自己的反馈及其状态，无法查看管理员评论。
- 删除操作不可恢复，请谨慎使用。
