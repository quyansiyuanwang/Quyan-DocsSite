# 验证页面

此页面用于处理登录、安全变更或高风险操作中的额外验证步骤。

## 页面用途

- 完成双重验证（2FA）。
- 通过 step-up 安全校验（高风险操作前二次确认）。
- 在常规二次验证不可用时使用恢复码。
- 完成验证码（Captcha）信任验证。

## 你会看到什么

### 验证方式

- 验证器应用动态码（TOTP）输入。
- 邮箱验证码输入。
- 恢复码输入（如支持）。
- 验证方式切换入口。

### Captcha 验证

在注册、发送验证码、找回密码、重置密码等操作前，系统可能要求完成 Captcha 验证：

- 支持 reCAPTCHA 和 Turnstile 两种提供商（由管理员在服务端配置页配置）。
- 通过验证后，系统在配置的信任窗口内不再重复弹出 Captcha。
- 可调用 `GET /auth/captcha/trust-status` 检查当前是否处于信任窗口期。

### 频率限制

系统对以下操作有按 IP / 邮箱维度的频率限制：

- 注册邮箱验证码发送。
- 找回密码验证码发送。
- 2FA 验证码验证（按 IP + 挑战令牌双维度）。
- 2FA 邮箱验证码发送。

超出限制时会收到 `429 Too Many Requests` 错误，并返回建议的重试等待时间。

### 操作区

- 提交按钮。
- 返回或取消按钮。
- 当前为什么需要验证的提示信息。

## 常见操作

1. 先阅读当前操作为何需要验证。
2. 选择可用的验证方式。
3. 输入对应验证码。
4. 提交后等待系统继续完成受保护操作。

## 说明

- 此页面可能在登录、关闭 2FA、邮箱变更或高风险操作（step-up）中出现。
- 恢复码应离线妥善保存，每套恢复码仅可使用一次。
- 如果页面中展示了政策同意内容，请先阅读后再继续。
- 防重放机制（Replay Protection）已在所有敏感表单上启用，刷新页面可获取新的签名会话。
